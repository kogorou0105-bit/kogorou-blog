# 笔记入口

这里放那些还不一定适合写成博客，但值得留下来的技术思考、项目记录和问题拆解。

博客更像一篇面向读者的完整文章，笔记更像工作现场里的整理：先把问题拆开，把链路记清楚，再慢慢沉淀成可复用的经验。

## 这里会放什么

- **技术笔记**：概念、工具、工作流、方法论，以及暂时还在推演中的想法。
- **项目记录**：一个项目从想法到实现的过程，包括取舍、调整和后续计划。
- **问题排查**：遇到问题时的定位过程、错误原因和下次可以复用的检查方法。

## 推荐写法

每篇笔记不用一开始就写得很完整，先写清楚几件事就够了：

- 背景：为什么要记录这件事？
- 问题：真正卡住的地方是什么？
- 链路：这件事从开始到结束是怎么流转的？
- 判断：现在能确定什么，还有什么需要继续验证？
- 沉淀：下次遇到同类问题，可以复用什么？

笔记可以保留问题感，不需要急着包装成结论。如果一篇笔记后来变得更完整、更适合对外表达，再整理成博客也不晚。
